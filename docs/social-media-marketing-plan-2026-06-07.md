## 二、社交媒体营销方案（现实版）

### 核心原则

**先把一个渠道跑通，再扩展。** 当前资源不应该同时开 5 个平台。

### 推荐优先级（Phase 顺序）

| 优先级 | 平台 | 原因 | 启动条件 |
|--------|------|------|---------|
| P0 | **YouTube** | 第二大搜索引擎，SEO 价值长期积累，案例视频可直接上传 | 已有真实案例视频 |
| P0 | **LinkedIn** | B2B 决策者聚集，FlarePix 目标客户（Amazon卖家 CEO/Founder/运营） | 账号已存在 |
| P1 | **TikTok** | 低成本测试，AI 产品视频题材适合病毒传播 | Phase 0完成后 |
| P2 | **Instagram** | 视觉平台，作品展示为主 | 有持续新作品时 |

**不做：** X/Twitter（欧美 B2B 效果差）、Facebook（目标用户不在这里）

---

### YouTube 执行计划

**为什么先做 YouTube：**
- 已有真实案例视频（GE、Arboleaf）可直接上传，无需额外制作
- YouTube 是全球第二大搜索引擎，长尾 SEO 价值持续积累
- 视频上传 YouTube 同时为网站带来直接引用流量

**YouTube 账号现状：** `youtube.com/@flarepix`（已创建，内容为空）

**执行步骤：**

```
第 1 周：
  1. 上传 2 条现有案例视频（GE 体重秤 + Arboleaf）
     - 添加详细描述（服务关键词 + 网站链接）
     - 添加 timestamp章节
     - 设置自定义缩略图（不要用 YouTube 自动生成的）
  2. 创建播放列表：
     - "Amazon Product Video Case Studies"
     - "Behind the Scenes — Hybrid Production"
  3. 完善 Channel Banner + Profile 图片
  4. 在 flarepix.com/blog 每篇文内嵌入对应 YouTube 视频

第 2 周：
  1. 上传第 3 条视频（如果有新的案例）
  2. 开始每周固定 1 条上传节奏
  3. 用博客文章标题作为 YouTube 视频标题（同一内容，SEO 双保险）
```

**YouTube 视频 SEO 规范（每条必做）：**

```
Title: [核心关键词] + [价值钩子] （≤60 字符）
  ✅ "GE Smart Scale Product Video — Amazon Listing Case Study"
  ❌ "Our GE project video"

Description:
  前 2 行：核心价值描述 + 关键词
  中间：时间戳章节（如果 >1 分钟）
  底部：flarepix.com → 服务页链接 + 相关博客链接
  标签：5-8 个关键词，从大到小

Thumbnail: 高对比度 + 大字 + 前后对比图
  尺寸：1280×720（16:9）
  文字：白色大字，黑边
```

## 三、视频内容自动化生产流水线（现实版）

### 重要前提：自动化是 Phase 3 的事，不是现在

你现在处于 Phase 0-1（广告没跑起来、外展没开始）。投入时间搭建自动化视频流水线会阻塞核心获客任务。

**推荐时序：**

```
Phase 0（现在）：   用现有案例视频跑 YouTube，不做任何自动化
Phase 1（2-4周后）：Google Ads 跑起来，有了询盘数据，再决定要不要做视频自动化
Phase 2（1-2月后）：如果 SEO 流量开始起来，内容需求变大，再考虑自动化流水线
```

### Phase 3 视频自动化流水线架构（参考用）

如果 Phase 1-2 跑通，内容需求变大时考虑：

| 层级 | 工具 | 成本 | 说明 |
|------|------|------|------|
| 选题层 | DataForSEO API（关键词挖掘）| $20/月 | 自动抓取 Amazon seller 相关搜索词 |
| 脚本层 | Claude API（脚本生成）| $5/月 | 输入：选题 + 平台 → 输出：结构化脚本 JSON |
| 素材层 | ElevenLabs（TTS）| $5-22/月 | 配音生成 |
| 合成层 | Python + FFmpeg（视频合成）| 自部署 | 将音频 + 画面素材合成为完整视频 |
| 发布层 | 手动发布到各平台 | 0 | 目前不建议自动化发布（平台政策不稳定）|

**最小可行流水线（MVP）：**

```
输入：选题（如 "5 Product Photography Mistakes Killing Your Amazon Sales"）
  ↓
Claude 生成脚本（JSON 结构）
  ↓
ElevenLabs 生成配音 + SRT 字幕
  ↓
FFmpeg 合成视频（背景图 + 配音 + 字幕 + logo 水印）
  ↓
输出：3 个尺寸（16:9 / 9:16 / 1:1）的手工精制视频
  ↓
手动发布到 YouTube / TikTok / LinkedIn
```

**自动化程度建议：**

| 部分 | 自动化？ | 原因 |
|------|---------|------|
| 关键词挖掘 | 半自动 | DataForSEO 给出词表，人工筛选 |
| 脚本生成 | 半自动 | AI 生成，人工审核（5 分钟/条）|
| TTS 生成 | 全自动 | ElevenLabs 直接生成 |
| 视频合成 | 全自动 | FFmpeg 脚本固定，无需人工 |
| 多尺寸输出 | 全自动 | 脚本固定，自动出 3 个尺寸 |
| 平台发布 | **手动** | 各平台政策不同，算法也在变，不建议自动发布 |

## 六、关键原则（执行纪律）

1. **先跑通一个渠道，再扩展** — 不要同时运营 YouTube + LinkedIn + TikTok + Instagram
2. **已有内容不要重新制作** — 8 篇博客 + 2 个案例视频是现有资产，直接复用
3. **自动化是后期的事** — Phase 3 才考虑流水线，现在用手工方式跑
4. **每周固定动作** — 每周一上传 1 条 YouTube，每周四发 1 条 LinkedIn Article
5. **数据追踪优先于内容生产** — GA4 + Google Sheet 跑起来之前，不要投入视频制作
